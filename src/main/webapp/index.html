Working with Queues & Backends

  1. Upload orders in xml format and enqueue them in a Push queue. The backend then processes the xml and creates orders.
  2. Confirm some orders for processing and enqueue them in a Pull Queue. It also updates the status to "processing" in the datastore.
  3. Process the confirmed orders; this triggers the backend instance for processing the tasks in pull queue. The backend consumes the task in queue and updates the status of orders to processed.

Sample Xml File format: There can be any number of orders which can be appended using the <order> tag

<?xml version="1.0" encoding="UTF-8" standalone="yes"?>
<orders>
  <order>
    <customer>cnam</customer>
    <status>pending</status>
    <user_orderid>rep1</user_orderid>
    <item>item1</item>
    <price>12</price>
    <quantity>12</quantity>
  </order>
  <order>
    <customer>cnam1</customer>
    <status>pending</status>
    <user_orderid>rep1</user_orderid>
    <item>item2</item>
    <price>120</price>
    <quantity>1</quantity>
  </order>
</orders>
